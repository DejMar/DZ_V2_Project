1. Pregled arhitekture

Projekt koristi troslojnu arhitekturu prilagođenu jednostavnim aplikacijama: prezentacija (UI), poslovna logika (Services) i podaci (JSON fajlovi). Nema baze podataka — umjesto nje koristi se generički JsonFileRepository.

Components/ — Blazor stranice (UI)
↓ poziva
Services/ — poslovna logika
↓ čita/piše
Data/*.json — perzistentni podaci

Models/ su zajedničke C# klase (DTO/entiteti) koje koriste i servisi i UI. Program.cs je ulazna tačka koja povezuje sve dijelove.

C# klase koje opisuju strukturu podataka. Koriste se u servisima i pri serijalizaciji u JSON.

FajlKlasaOpis

UserRole.cs UserRole Enum: Administrator, Moderator, Korisnik
Medicine.cs Medicine Lijek: naziv, opis, količina, jedinica, min. zaliha, expiryDate. Svojstva: IsLowStock, IsExpired, IsExpiringSoon
Ambulance.cs Ambulance Ambulanta u domu zdravlja (id, naziv)

StockIntake.cs StockIntake Evidentirani prijem zaliha: lijek, količina, datum, ko je uneo, napomena, opcionalni novi rok trajanja
DashboardSummary.cs DashboardSummary, DashboardAlert, itd. Modeli za dashboard: obavijesti, statistika, kritični lijekovi, zahtjevi, prijemi (ne čuvaju se u JSON)

5. Services/ — poslovna logika

Servisi enkapsuliraju logiku i komuniciraju sa JSON fajlovima. Registruju se u Program.cs.

JsonFileRepository.cs
Generička klasa JsonFileRepository<T> — čita i piše listu objekata u JSON fajl. Metode: GetAllAsync(), SaveAllAsync(). Automatski kreira Data/ folder ako ne postoji.
Koriste: svi ostali servisi
DataInitializer.cs
Pri prvom pokretanju aplikacije popunjava prazne JSON fajlove demo podacima (5 korisnika, 3 ambulante, 15 lijekova sa rokom trajanja, prazan stock_intakes.json). MigrateLegacyUsersAsync postavlja isActive=true za stare JSON fajlove.
Poziva: Program.cs pri startu
AuthService.cs
Upravlja trenutno prijavljenim korisnikom. Čita korisnika iz ASP.NET session-a (EnsureLoadedAsync). Metode: IsAuthenticated, IsInRole.
Koriste: AuthGuard, MainLayout, SidebarFooter, sve zaštićene stranice
MedicineService.cs
CRUD za lijekove: dodavanje, uređivanje, brisanje, smanjenje zalihe pri izdavanju. Radi sa medicines.json.
Koriste: Medicines.razor, RequestService, ReportService, NewRequest.razor
AmbulanceService.cs
Čitanje ambulanti i dohvat naziva po ID-u. Radi sa ambulances.json.
Koriste: NewRequest.razor, Requests.razor, ReportService
RequestService.cs
Upravljanje zahtjevima: kreiranje, odobravanje, odbijanje, izdavanje. DeliverAsync blokira izdavanje isteklih lijekova i vraća poruku greške. Pri izdavanju poziva MedicineService.ReduceStockAsync. Radi sa requests.json.
Koriste: NewRequest.razor, MyRequests.razor, Requests.razor, ReportService
UserService.cs
CRUD za korisnike: dodavanje, uređivanje, provjera jedinstvenog username-a, aktivacija/deaktivacija naloga. Radi sa users.json.
Koriste: Users.razor, AuthEndpoints (login provjera)
ReportService.cs
Generiše ReportSummary za admin izvještaje: broj lijekova, niske zalihe, istekli/uskoro ističući lijekovi, statistika zahtjeva po statusu i po ambulantama.
Koriste: Reports.razor, ReportExportService
ReportExportService.cs
Generiše izvoz izvještaja: GenerateCsvAsync() za CSV fajl i GenerateHtmlReportAsync() za štampani HTML izvještaj.
Koristi: ExportEndpoints.cs
StockIntakeService.cs
Evidentiranje prijema zaliha: RecordIntakeAsync() snima zapis i povećava zalihu lijeka. Opcionalno ažurira rok trajanja. Radi sa stock_intakes.json.
Koriste: PrijemZaliha.razor, DashboardService
DashboardService.cs
Generiše DashboardSummary po ulozi: admin (kritični lijekovi, pending, prijemi), moderator (zahtjevi na čekanju, stariji od 2 dana, odobreni), korisnik (statistika vlastitih zahtjeva).
Koristi: Dashboard.razor

7. Ostali važni fajlovi

FajlOpis
Program.cs Ulazna tačka aplikacije. Registruje servise (DI container), konfiguriše session, pokreće DataInitializer, mapira auth/export endpointe i Blazor rute.
AuthEndpoints.cs HTTP endpointi za prijavu (POST /auth/login) i odjavu (GET /auth/logout). Login provjerava isActive i postavlja session cookie.
ExportEndpoints.cs Izvoz izvještaja za admina: GET /admin/export/csv (CSV download) i GET /admin/export/report (HTML za štampu/PDF). Zaštićeno session provjerom uloge Administrator.
DomZdravlja.csproj Definicija .NET 8 web projekta — target framework, SDK tip.
appsettings.json Opšta konfiguracija (logovanje nivoi). Nema connection stringa jer nema baze.
appsettings.Development.json Dodatna konfiguracija samo za development okruženje.
Properties/launchSettings.json Port (5141), HTTPS profil, environment varijable pri pokretanju iz IDE-a.
wwwroot/app.css Globalni CSS: boje, layout, tabele, forme, dugmad, login stranica, responsive stilovi.
Data/*.json Runtime podaci. Kreiraju se automatski. Svaki fajl = jedna kolekcija entiteta. Nisu u git-u (vidi .gitignore).
DomZdravlja.sln Visual Studio solution — grupiše projekat za otvaranje u IDE-u.
.gitignore Isključuje bin/, obj/ i Data/ iz verzionisanja.

9. Kako fajlovi komuniciraju — primjeri toka

Prijava korisnika

Login.razor → POST forma → AuthEndpoints.cs (/auth/login) → čita users.json preko JsonFileRepository<User> → postavlja session → redirect na /pregled → AuthService.EnsureLoadedAsync() učitava korisnika iz session-a na svakoj stranici

Admin/Moderator evidentira prijem zaliha

PrijemZaliha.razor → StockIntakeService.RecordIntakeAsync() → snima u stock_intakes.json → MedicineService.UpdateAsync() povećava zalihu u medicines.json

Korisnik otvara dashboard

Dashboard.razor → DashboardService.Get*DashboardAsync() → agregira podatke iz medicines, requests, stock_intakes → vraća DashboardSummary sa obavijestima po ulozi

Korisnik šalje zahtjev

NewRequest.razor → RequestService.CreateAsync() → JsonFileRepository<MedicationRequest> → requests.json

Moderator izdaje lijek

Requests.razor → RequestService.DeliverAsync() → provjera Medicine.IsExpired → ako istekao, vraća grešku → MedicineService.ReduceStockAsync() → medicines.json → ažurira status u requests.json

Admin upravlja korisnicima

Users.razor → UserService.AddAsync() / UpdateAsync() / SetActiveAsync() → JsonFileRepository<User> → users.json

Admin izvozi izvještaj

Reports.razor → link na ExportEndpoints.cs → ReportExportService → ReportService.GetSummaryAsync() → generiše CSV ili HTML fajl za preuzimanje

Admin pregleda izvještaj

Reports.razor → ReportService.GetSummaryAsync() → čita medicines.json + requests.json + ambulances.json → vraća ReportSummary → prikazuje na stranici

Registracija servisa (dependency injection)

U Program.cs servisi se registruju sa različitim životnim ciklusima:

Tip registracijeServisiZnačenje
Singleton JsonFileRepository, DataInitializer Jedna instanca za cijelu aplikaciju (dijele pristup fajlovima)
Scoped AuthService, UserService, MedicineService, RequestService, ReportExportService, StockIntakeService, DashboardService, itd. Nova instanca po HTTP zahtjevu / Blazor circuit-u
